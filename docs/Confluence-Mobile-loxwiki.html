    Java web interface

    Bouni

    May 03, 2017

    Watch Watc h 

     

    Before firmware version 3.81, the Luxtronik 2 delivered a Java applet as a web interface, which read and write data via port 8888 or 8889.

    From version 3.81, a protocol based on a web socket was changed.

    The protocol previously used by the Java applet is still active! and is used, for example, by the smartphone app.

    Web interface

    The web interface itself is unsuitable to be addressed using MS, but this must somehow also come from the heating data , which we can use for our purposes.

    The web interface can be reached at the IP address of the heat pump, e.g. 192.168.178.10. If you look at the Ethernet packets that go to and from the heat pump after the web interface has been called up (middle Wireshark ), you can see that the web interface opens a new connection on port 8889 to the WP (8888 on Luxtronik 2 modules with SW versions < 1.76). The Java applet sends certain byte sequences to the WP and then gets a whole series of bytes back. This behavior can be simulated with the PicoC functionality of the MS and you get the desired information. 

    Establish connection and request data

    To set up a connection, connect to the IP of the Luxtronik using port 8889.

    There are 4 possible commands that can be sent to the Luxtronik.

    The translations of the names may not be correct!

    3002 - Write parameters

    Send 4 bytes with the command, Big Endian formatted, followed by 4 bytes with the parameter number and 4 bytes with the new value:

    0x00 0x00 0x0b 0xba 0x00 0x00 0x00 0x00 0x00 0x00 0x00 0x00

    The Luxtronik responds with the command (4 bytes) and the parameter value (4 bytes) , also formatted as Big Endian. 

    3003 - Read parameters

    You send 4 bytes with the command, Big Endian formatted, followed by 4 zero bytes:

    0x00 0x00 0x0b 0xbb 0x00 0x00 0x00 0x00

    The Luxtronik responds with the command (4 bytes) and the number of parameters that follow (4 bytes) , also Big Endian formatted. 

    As soon as you know the number, you read this number of integers (4 bytes each) 

    Description: 3003.txt 

    3004 - Read calculations

    You send 4 bytes with the command, Big Endian formatted, followed by 4 zero bytes:

    0x00 0x00 0x0b 0xbc 0x00 0x00 0x00 0x00

    The Luxtronik replies with the command (4 bytes), the status (4 bytes) and the number of calculations that follow (4 bytes) , also Big Endian formatted.  

    As soon as you know the number, you read this number of integers (4 bytes each) 

    Below is a table that describes (almost) all parameters and shows how to interpret the values.

    •                                 Fields highlighted in red are unclear
    •                                 The parameters 0 - 10 are also unclear, the parameters from 10 are only dealt with in the Java code
    •                                 The parameters 6 + 7 give me a Unix timestamp that can be converted into "01.01.2007 01:00:00", but the meaning is not clear
    •                                 Many parameters are with me with 0, I suspect that the hardware is not built into the data anyway to kill all models with software.

    Integer to IP

    void print_ip (int ip) {

        unsigned char bytes [4];

        bytes [0] = ip & 0xFF;

        bytes [1] = (ip >> 8) & 0xFF;

        bytes [2] = (ip >> 16) & 0xFF;

        bytes [3] = (ip >> 24) & 0xFF;

    3005 - Read Visibilities

    You send 4 bytes with the command, Big Endian formatted, followed by 4 zero bytes :

    0x00 0x00 0x0b 0xbd 0x00 0x00 0x00 0x00

    The Luxtronik answers with the command (4 bytes) and the number of calculations that follow (4 bytes) , also Big Endian formatted. 

    As soon as you know the number, you read this number of bytes (attention, not 4 bytes!) 

    Description: 3005.txt
